2.1 Review the ongoing use of IT tools and change approach as needed – in making your own website, are you happy with the tools you used? E.g. Visual Studio Code, Trello

Trello helped me to plan and organise the project. I was able to track the progress of the project and email notifications from Trello reminded me of deadlines when I was not using trail. I used visual studio code to code and design the website using languages like HTML and CSS. I also used google for other resources such as google fonts which were part of the coding of the website.

2.2 Describe whether the IT tools selected were appropriate E.g. Visual Studio Code, Trello

As stated previously visual studio code was the platform used for coding and designing of the website by incorporating open-end programmes such as HTML and CSS. Trello was the tool used to organise and track the progress of the project. Google was used to research questions which were part of the project. However as an individual new to coding I felt that I needed more time to understand the languages such as html and css better and that would have allowed me to optimise their use efficiently.

2.3 Assess the strengths and weaknesses of your final project – are you happy with your website? Why? Why not? You should review it on the website itself in the interests of openness. What would you change?

If I had more time to design the website I would have ensured that my website was complex (fancy features) simultaneously retaining user friendly capability. I would have loved to experiment and play around with other features like adding GIFs and animations to make my website exciting and attract more users. I would also have conducted more research on website design to borrow some ideas from other website.

2.4 Describe further improvements you can make to your project

As stated earlier, I would have loved more time to design the website to have more colours and buttons, images, videos and animations. Just to experience with more ideas and codes to come up with the the best design., better layout, best possible colour combination, use of flexbox.
